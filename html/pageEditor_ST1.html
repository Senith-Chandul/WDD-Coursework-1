Splash Screen

Introduction:

The Splash Screen serves as the introductory page for the website, displaying the team members and redirecting users to the home page after 5 seconds. It features a clean design with a loader animation to indicate the redirect process.

Technical Description

HTML Techniques:

The `splash.html` page uses a meta refresh tag (<meta http-equiv="refresh">) for automatic redirection. Semantic HTML tags like <header> and <section> structure the content. Team members are listed using an unordered list (<ul>).

CSS Techniques:

The `style.css` file styles the splash screen with a centered layout and semi-transparent background. Key features include:

  • Loader Animation – Created using CSS keyframes for rotation effects.
  • Responsive Typography – Font sizes adjust for different screen sizes using media queries.
  • Gradient Background – Subtle overlay on the background image for text readability.

Purpose and Logic:

The splash screen provides a professional first impression while ensuring seamless navigation to the main website. The 5-second delay allows users to view team information before redirection.

Challenges and Lessons Learned:

Ensuring the loader animation synchronized perfectly with the 5-second redirect required precise timing adjustments. Testing across browsers revealed inconsistencies in meta refresh behavior, which were resolved using JavaScript as a fallback.

Accessibility:

The page uses ARIA labels for the loader animation and semantic HTML for screen readers. Color contrast ratios meet WCAG standards.

Compliance:

Complies with Janet regulations through proper redirection logic and accessible content structuring. Passes W3C validation for HTML5 and CSS3.

References:

1. W3C (n.d.) Web Content Accessibility Guidelines (WCAG). Available at: https://www.w3.org/WAI/

Volunteer Page

Introduction:

The Volunteer Page allows users to explore marine conservation programs and submit feedback. It features interactive program cards, a feedback form, and user testimonials.

Technical Description

HTML Techniques:

Built using CSS Grid for program cards (<div class="programs-container">) and Flexbox for the feedback form. The form includes input fields for name, email, rating, and comments.

CSS Techniques:

Key features in `volunteer.css`:

  • Hover Effects – Program cards scale on hover using transform: scale()
  • Overlay Animation – Details fade in on card hover using opacity transitions
  • Responsive Grid – Cards reflow for mobile using grid-template-columns: repeat(auto-fit, minmax())

JavaScript Features:

Implemented scroll-triggered fade-in animations using Intersection Observer API. Form validation ensures all required fields are completed.

Challenges and Lessons Learned:

Aligning the overlay text across different card sizes required position: absolute with top/bottom/left/right: 0. Testing revealed z-index issues in mobile view, resolved through careful stacking context management.

Accessibility:

Form fields include proper <label> associations. ARIA roles added for screen readers. Color contrast meets AA standards.

Compliance:

Passes W3C validation with zero errors. Complies with GDPR through explicit consent checkboxes in the form.

Content Page

Introduction:

The Sustainable Fishing content page educates users about responsible fishing practices through structured sections, image galleries, and interactive elements.

Technical Description

HTML Techniques:

Uses semantic <section> tags with ID-based navigation. Image galleries employ <div class="content-gallery"> with Flexbox layout. The sticky navigation bar uses position: sticky.

CSS Techniques:

Key features in `content.css`:

  • Sticky Navigation – position: sticky with top: 0
  • Responsive Images – object-fit: cover maintains aspect ratios
  • Section Transitions – Smooth scroll behavior using scroll-behavior: smooth

JavaScript Features:

Implemented active state highlighting in the navigation bar using scroll event listeners. The "Go to Top" button appears on scroll using CSS transitions.

Challenges and Lessons Learned:

Balancing text density with visual elements required careful grid planning. Initially poor Lighthouse scores for CLS were improved by setting explicit image dimensions.

Accessibility:

All images include descriptive alt text. Navigation links have aria-current="page" indicators. Reduced motion preferences respected in animations.

Compliance:

Meets WCAG 2.1 AA standards. Schema.org microdata added for better SEO.
