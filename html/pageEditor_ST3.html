Feedback Page

Introduction:

The Feedback Page is designed to collect user opinions and suggestions to improve the "Beneath the Waves" website. It features a user-friendly form with fields for personal details, feedback, and ratings, including interactive elements like emoji ratings and dropdown menus. Built with HTML for structure and CSS for styling, the page ensures accessibility and responsiveness, making it easy for users to share their thoughts and help enhance the platform.

Technical Description

HTML Techniques:

The `feedback.html` page is structured using semantic HTML tags to ensure clarity and accessibility. The <form> element is used to create a feedback form, with <fieldset> and <legend> tags to group related form fields and provide context. Input fields such as <input>, <textarea>, and <select> are used to collect user data. The <label> tags are associated with each input for better accessibility. The form includes various input types like text, email, tel, date, and file, along with radio buttons and checkboxes for multiple-choice questions. The <span> tags are used to display error messages for invalid inputs.

CSS Techniques:

The `feedback.css` file styles the feedback form to create a visually appealing and user-friendly interface. The `.feedback-form` class centers the form and applies a semi-transparent background with rounded corners and a shadow effect. The `fieldset` and `legend` elements are styled with borders and colors to differentiate sections. Input fields and textareas are styled uniformly with padding, borders, and background colors to maintain consistency. The `.radio-group` and `.checkbox-group` classes use Flexbox to align options horizontally and vertically, respectively. The `.emoji-rating` class uses Flexbox to display emojis in a row, with hover effects to enhance interactivity. Error messages are styled with red borders and text to indicate invalid inputs. The `.submit-button` class styles the submit button with a background color transition on hover for a smooth visual effect.

Purpose and Logic:

The HTML structure ensures the form is accessible and easy to navigate, while the CSS styling enhances the user experience with a cohesive design and interactive elements. The use of semantic tags and proper labeling improves accessibility, and the styling choices create a modern, engaging interface that encourages user interaction.

Key HTML Tags and CSS Properties:

1. HTML Tags
  • <form>, <fieldset>, <legend>: Form structure and grouping.
  • <input>, <textarea>, <select>: Form inputs.
  • <label>: Labels for form fields.
  • <span>: Emojis for rating.
2. CSS Properties:
  • display: flex;: Aligns radio buttons, checkboxes, and emojis.
  • background-color: rgba();: Semi-transparent background for the form.
  • border-radius: 10px;: Rounded corners for form sections.
  • transform: scale();: Hover effects for emojis.
  • transition: background-color 0.3s ease;: Smooth hover effect for buttons.

Challenges and Lessons Learned:

During the implementation of the feedback form, one challenge was ensuring proper form validation to prevent incomplete submissions. This was addressed using JavaScript validation, which highlights missing fields and provides alerts. Another challenge was improving accessibility, which was resolved by adding <label> tags for all input fields. These experiences reinforced the importance of user-friendly and error-proof form design.

Accessibility:

The feedback page follows accessibility best practices by using semantic HTML elements like <fieldset>, <legend>, and <label> to improve screen reader compatibility. The form includes required attributes to ensure users complete necessary fields, and radio buttons, checkboxes, and dropdowns are clearly labeled for ease of navigation. Additionally, contrast and spacing considerations help users with visual impairments.

Compliance:

The page aligns with web accessibility and usability standards by following Janet regulations for web publication. It ensures proper structuring of content, clear labeling of form fields, and the use of valid HTML5 elements. Input validation prevents incorrect data submissions, and privacy considerations are maintained by keeping user data collection minimal and relevant.

Team Page

Introduction:

The Team Page showcases the members of the "Beneath the Waves" initiative, highlighting their roles and contributions to the project. Each team member is presented in a visually appealing card layout, featuring their photo, name, role, and a brief description of the pages they designed and implemented. The page uses a responsive grid layout to ensure it looks great on all devices, with hover effects that add interactivity. The design is clean and modern, with a focus on accessibility and user engagement, making it easy for visitors to learn about the team and navigate to individual editor pages for more details.

Technical Description

HTML Techniques:

1. Semantic Structure:
  • The page uses semantic HTML tags like <header>, <main>, and <footer> to structure the content logically. This improves accessibility and SEO.
  • The <nav> element is used for the navigation bar, which contains links to other pages on the website.
2. Grid Layout for Team Members:
  • The team members are displayed in a grid layout using the <div class="team-members"> container. Each team member is represented by an <a> tag, which links to their respective editor's page.
  • The grid layout is achieved using CSS Grid (display: grid;), which allows for responsive and flexible positioning of team members.
3. Image Backgrounds:
  • Each team member's image is set as a background image using inline CSS (style="background-image: url('../assets/member3.jpg');"). This allows for easy customization and ensures the images are responsive.
4. Accessibility:
  • The <a> tags for team members include descriptive text (name, role, and responsibilities) to ensure screen readers can interpret the content correctly.
  • The alt attribute is not used here since the images are background images, but the text content inside the <a> tags provides sufficient context.

Css Techniques:

1. Responsive Grid Layout:
  • The .team-members class uses grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); to create a responsive grid. This ensures that the team members are displayed in a flexible layout that adjusts based on the screen size.
  • The gap: 2rem; property adds spacing between grid items, improving readability and visual appeal.
2. Hover Effects:
  • The .team-member:hover class applies a transform (transform: translateY(-10px);) and changes the background color (background-color: var(--bright-blue);) when a user hovers over a team member. This provides visual feedback and enhances interactivity.
3. Image Styling:
  • The .member-image class sets the height of the image container to 400px and uses background-size: cover; and background-position: center; to ensure the images are displayed correctly and maintain their aspect ratio.
4. Typography and Colors:
  • The .team-member h3 and .team-member p classes are used to style the text inside each team member card. The color property is used to ensure the text contrasts well with the background.
  • The #role selector is used to style the role description with a larger font size (font-size: 1.1rem;) and bold text (font-weight: 600;).
5. Box Shadows and Borders:
  • The .team-member class uses box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1); to add a subtle shadow effect, giving the cards a raised appearance.
  • The border-radius: 10px; property rounds the corners of the team member cards, making them visually appealing.

Purpose and Logic:

  • Responsive Design – The grid layout ensures that the team members are displayed in a flexible and responsive manner, adapting to different screen sizes. This is crucial for providing a good user experience on both desktop and mobile devices.
  • Interactivity – The hover effects on the team member cards provide visual feedback, making the page more engaging and interactive.
  • Accessibility – The use of semantic HTML and descriptive text ensures that the page is accessible to all users, including those using screen readers.
  • Visual Appeal – The combination of box shadows, rounded corners, and hover effects creates a modern and visually appealing design that aligns with the overall theme of the website.

Key HTML Tags and CSS Properties:

1. HTML Tags
  • <header>, <main>, <footer>: Semantic structure
  • <nav>: Navigation bar.
  • <a>: Links to team member editor pages.
  • <div>: Container for team members and images.
  • <h3>, <p>: Text content for team member details.
2. CSS Properties:
  • display: grid;: Creates a grid layout for team members
  • grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));: Responsive grid columns.
  • background-image: url();: Sets background images for team members.
  • transform: translateY();: Adds hover effect.
  • box-shadow: 0 4px 8px rgba();: Adds shadow to team member cards.
  • border-radius: 10px;: Rounds the corners of the cards.

Challenges and Lessons Learned:

One challenge encountered was ensuring consistent styling across multiple pages while integrating different designs by team members. This was resolved by creating a common CSS file (style.css) to standardize fonts, colors, and layout structure. Additionally, linking separate CSS files (team.css) allowed for customization without affecting other pages. Another challenge was maintaining smooth navigation, which was addressed by implementing a clear and structured navigation bar across all pages.

Accessibility:

The Team page follows accessibility principles by using semantic HTML elements (<header>, <main>, <footer>) for better screen reader support. Images are used as backgrounds for visual aesthetics, but alternative text (alt) should be added to enhance accessibility. The navigation bar provides clear labels for each section, and the text is structured with appropriate headings (<h1>, <h3>) to improve readability.

Compliance:

The Team page complies with Janet regulations by ensuring proper navigation, content clarity, and structure. The page follows web publishing standards by maintaining clear labeling of team roles, ensuring structured hyperlinks, and keeping an organized footer with relevant editor links. Additionally, the navigation bar provides logical, accessible, and user-friendly browsing, which aligns with compliance guidelines.

Content Page

Introduction:

The Content Page (Sustainable Fishing) was designed using structured HTML and CSS to ensure accessibility and responsiveness. Key elements in HTML include semantic tags like <header>, <main>, and <footer> to structure the content. The CSS Grid and Flexbox techniques were used for layout management, ensuring a visually appealing and mobile-friendly design. Navigation bars use unordered lists (<ul>) with CSS styling for horizontal menus. Image sections were structured using <div> containers with background-image and <img> tags for responsive visuals. The "Go to Top" button was implemented as an anchor link for smooth navigation.

Challenges and Lessons Learned:

One challenge was ensuring consistent styling across different pages while maintaining unique elements for each. This was resolved by creating separate CSS files for each page while keeping common styles in a shared style.css. Additionally, handling responsive images was a challenge, but using max-width: 100% and object-fit: cover helped maintain proportion. The key lesson learned was the importance of modular CSS structuring and planning the HTML layout before implementation

Accessibility:

The pages were built with accessibility in mind, using semantic HTML elements (<nav>, <section>, <h1>-<h3>) to help screen readers navigate the content. Alt text (alt="") was provided for all images to support visually impaired users. Sufficient color contrast was used in the text for readability, and the navigation bar follows a logical tab order, allowing keyboard navigation.

Compliance:

The web pages comply with Janet regulations by ensuring structured HTML, ethical content, and accessibility standards. The use of valid HTML5 elements, clear labeling, and responsive design aligns with best practices. There is no misleading information, and all images used are relevant and properly sourced.

Profile Page

Introduction:

The Profile Page is designed to showcase the personal and professional details of a user involved in the "Beneath the Waves" initiative. It features a clean and modern layout, highlighting the user's avatar, role, and a brief description of their passion for marine conservation. The page also includes sections for volunteering activities, skills and goals, availability, and a contact form, making it a comprehensive hub for user information. With a responsive design and interactive elements like progress bars and hover effects, the Profile Page ensures an engaging and user-friendly experience while maintaining a focus on accessibility and visual appeal.

Technical Description

HTML Techniques:

1. Semantic Structure:
  • The page uses semantic HTML tags like 'header', 'main', and 'section' to structure the content logically. This improves accessibility and SEO..
  • The 'nav' element is used for the navigation bar, which contains links to other pages on the website.
2. User Avatar and Information Section:
  • The user's profile information is displayed using a combination of 'div', 'img', and 'p' tags. The 'img' tag is used to display the user's avatar, while 'h2' and 'p' tags are used for the user's name, role, and description.
  • The 'ul' and 'li' tags are used to list the user's volunteering activities.
3. Skills & Goals Section:
  • The skills and goals are displayed using an unordered list ('ul'), with each skill represented by a 'li' tag. A 'span' element with a progress bar class is used to visually represent the progress of each skill.
4. Calendar Section:
  • The availability calendar is created using a 'table' element, with 'thead' and 'tbody' for structuring the days and times. This provides a clear and organized way to display the user's schedule.
5. Contact Form:
  • The contact form is created using the 'form' element, with 'input' and 'textarea' tags for user input. Labels are associated with each input field using the 'label' tag, improving accessibility.

Css Techniques:

1. Flexbox Layout:
  • The .profile-main class uses display: flex; to create a flexible layout for the profile sections. This ensures that the content is evenly distributed and responsive.
  • The .interest-icons class uses display: flex; and justify-content: center; to align the interest icons horizontally.
2. Avatar Styling:
  • The .avatar img class styles the user's avatar with a circular shape (border-radius: 50%;) and a border (border: 5px solid var(--bright-blue);). This creates a visually appealing and professional look.
3. Progress Bars:
  • The .progress-bar class is used to create progress bars for the skills section. The width property is dynamically set using inline styles (style="width: 80%;") to represent the progress percentage.
  • The ::after pseudo-element is used to display the percentage value next to the progress bar.
4. Table Styling:
  • The .calendar table class styles the calendar table with borders (border: 1px solid var(--light-blue);) and padding (padding: 0.5rem;). This ensures the table is easy to read and visually consistent with the rest of the page.
5. Form Styling:
  • The .contact-me form class uses flex-direction: column; to stack the form inputs vertically. The input and textarea elements are styled with padding, borders, and background colors to ensure they are visually appealing and easy to use.
  • The button element is styled with a background color transition (transition: background-color 0.3s ease;) to provide visual feedback when hovered over.

Purpose and Logic:

  • Responsive Design – The use of Flexbox ensures that the profile page is responsive and adapts to different screen sizes, providing a consistent user experience across devices.
  • Visual Hierarchy – The styling of headings (<h2>, <h3>), progress bars, and the avatar creates a clear visual hierarchy, making it easy for users to scan and understand the content.
  • Interactivity – The hover effects on the progress bars and buttons enhance interactivity, providing visual feedback to users as they interact with the page.
  • Accessibility – The use of semantic HTML tags and proper labeling of form inputs ensures that the page is accessible to all users, including those using screen readers.

Key HTML Tags and CSS Properties:

1. HTML Tags
  • <header>, <main>, <section>: Semantic structure
  • <img>: User avatar.
  • <ul>, <li>: Lists for volunteering activities and skills.
  • <table>, <thead>, <tbody>: Calendar layout.
  • <form>, <input>, <textarea>, <label>: Contact form.
2. CSS Properties:
  • display: flex;: Creates flexible layouts for sections and icons.
  • border-radius: 50%;: Styles the avatar as a circle.
  • width: 80%;: Dynamically sets the width of progress bars.
  • ::after: Displays percentage values next to progress bars.
  • border: 1px solid var(--light-blue);: Styles table borders.
  • transition: background-color 0.3s ease;: Adds hover effects to buttons.
