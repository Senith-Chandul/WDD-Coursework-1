Template

Introduction:

The Template serves as the foundation for all pages on the "Life Below Water" website. It includes a consistent header, navigation bar, and footer across all pages, ensuring a cohesive user experience. The template is designed to be responsive, accessible, and visually appealing, with a focus on reusability and modularity.

Technical Description

HTML Techniques:

The template is built using semantic HTML tags like <header>, <nav>, <main>, and <footer>. The navigation bar is created using an unordered list (<ul>) with list items (<li>) and anchor tags (<a>) for each page link. The footer contains copyright information and links to editor pages.

CSS Techniques:

The `style.css` file styles the template with a fixed header, responsive navigation bar, and a footer that sticks to the bottom of the page. Flexbox is used to align the navigation links horizontally, and media queries ensure the layout adapts to different screen sizes. The header and footer use a dark blue background color with light blue text for contrast.

Purpose and Logic:

The template ensures consistency across all pages by providing a reusable structure for the header, navigation bar, and footer. The responsive design ensures the website is accessible on all devices, while the modular structure makes it easy to add or modify content.

Key HTML Tags and CSS Properties:

  • <header>, <nav>, <footer>: Semantic structure.
  • <ul>, <li>, <a>: Navigation bar.
  • display: flex;: Horizontal alignment of navigation links.
  • position: fixed;: Fixed header.
  • background-color: var(--very-dark-blue);: Header and footer styling.

Challenges and Lessons Learned:

One challenge was ensuring the navigation bar remained responsive on smaller screens. This was resolved by using media queries to adjust the font size and padding. Another challenge was making the footer stick to the bottom of the page, which was achieved using Flexbox and min-height properties.

Accessibility:

The template follows accessibility best practices by using semantic HTML tags and providing clear labels for navigation links. The navigation bar is keyboard-friendly, and the footer includes descriptive text for screen readers.

Compliance:

The template complies with Janet regulations by ensuring proper content structuring, clear labeling, and responsive design. It also adheres to WCAG guidelines for accessibility.

Navigation Bar

Introduction:

The Navigation Bar is a key component of the template, providing users with easy access to all major pages on the website. It is designed to be responsive, accessible, and visually consistent across all pages. The navigation bar includes links to the Home, Impact Data, Content, and Sitemap pages, as well as other relevant sections.

Technical Description

HTML Techniques:

The navigation bar is created using an unordered list (<ul>) with list items (<li>) and anchor tags (<a>) for each page link. The active page is highlighted using a CSS class (`active`), and hover effects are applied to enhance interactivity.

CSS Techniques:

The `style.css` file styles the navigation bar with a dark blue background color and light blue text. Flexbox is used to align the links horizontally, and hover effects change the text color to bright blue. The active page is highlighted with a border-bottom and a different text color.

Purpose and Logic:

The navigation bar provides a consistent and intuitive way for users to navigate the website. The responsive design ensures it works well on all devices, while the hover and active states enhance user interaction.

Key HTML Tags and CSS Properties:

  • <ul>, <li>, <a>: Navigation bar structure.
  • display: flex;: Horizontal alignment of links.
  • color: var(--light-blue);: Text color.
  • border-bottom: 2px solid var(--bright-blue);: Active page indicator.

Challenges and Lessons Learned:

One challenge was ensuring the navigation bar remained usable on smaller screens. This was resolved by using media queries to stack the links vertically and adjust the font size. Another challenge was implementing the active page indicator, which was achieved using JavaScript to dynamically add the `active` class.

Accessibility:

The navigation bar follows accessibility best practices by using semantic HTML tags and providing clear labels for links. It is keyboard-navigable, and the active page is clearly indicated for screen readers.

Compliance:

The navigation bar complies with Janet regulations by ensuring proper content structuring, clear labeling, and responsive design. It also adheres to WCAG guidelines for accessibility.

Home Page

Introduction:

The Home Page serves as the main landing page for the "Life Below Water" initiative. It introduces users to the website's mission and provides quick access to key focus areas such as marine pollution, coral reef protection, sustainable fishing, and climate impact. The page features a hero section with a call-to-action, followed by interactive tiles that link to content pages. The design is clean, modern, and responsive, ensuring a seamless user experience across devices.

Technical Description

HTML Techniques:

The `home.html` page uses semantic HTML tags like <header>, <main>, and <section> to structure the content. The hero section is created using a <div> with a background image and overlay. The focus areas are displayed using a grid layout with <a> tags wrapped around <div> elements for each tile. Each tile contains an image and an overlay with text.

CSS Techniques:

The `style.css` file styles the home page with a responsive grid layout for the tiles, hover effects for interactivity, and a fixed navigation bar. The hero section uses a linear gradient overlay for better text visibility. Flexbox and CSS Grid are used to ensure the layout adapts to different screen sizes.

Purpose and Logic:

The home page is designed to be visually engaging and user-friendly. It provides a clear overview of the website's purpose and directs users to explore specific topics. The responsive design ensures accessibility on all devices, while the interactive elements encourage user engagement.

Key HTML Tags and CSS Properties:

  • <header>, <main>, <section>: Semantic structure.
  • <div>: Container for hero section and tiles.
  • display: grid;: Creates a responsive grid layout for tiles.
  • background-image: linear-gradient();: Adds overlay to the hero section.

Challenges and Lessons Learned:

One challenge was ensuring the hero section's text remained readable across different screen sizes. This was resolved by using a combination of responsive font sizes and a gradient overlay. Another challenge was creating a seamless transition between the hero section and the tiles, which was achieved by adjusting the padding and margins.

Accessibility:

The home page follows accessibility best practices by using semantic HTML tags and providing alt text for images. The navigation bar is keyboard-friendly, and the tiles are clearly labeled for screen readers.

Compliance:

The home page complies with Janet regulations by ensuring proper content structuring, clear labeling, and responsive design. It also adheres to WCAG guidelines for accessibility.

Table Page

Introduction:

The Table Page displays impact data related to marine conservation efforts across various countries. It uses a responsive table layout to present data on CO₂ emissions, protected areas, and conservation status. The table is styled with icons and color-coded status indicators to enhance readability and user engagement.

Technical Description

HTML Techniques:

The `table.html` page uses the <table> element to structure the data. Each row (<tr>) contains cells (<td>) for country names, CO₂ emissions, protected areas, and status. Icons are added using Font Awesome, and status indicators are styled with <span> tags.

CSS Techniques:

The `table.css` file styles the table with alternating row colors, hover effects, and responsive design. The status indicators use background colors and border-radius to create a pill-like appearance. The table is centered on the page and adapts to different screen sizes.

Purpose and Logic:

The table page provides a clear and concise way to present complex data. The use of icons and color-coded status indicators makes it easy for users to quickly understand the information. The responsive design ensures the table is accessible on all devices.

Key HTML Tags and CSS Properties:

  • <table>, <tr>, <td>: Table structure.
  • <span>: Status indicators.
  • background-color: rgba();: Alternating row colors.
  • border-radius: 15px;: Pill-like status indicators.

Challenges and Lessons Learned:

One challenge was ensuring the table remained readable on smaller screens. This was resolved by using horizontal scrolling for the table and adjusting font sizes. Another challenge was styling the status indicators, which was achieved using CSS pseudo-elements.

Accessibility:

The table page follows accessibility best practices by using semantic HTML tags and providing clear labels for icons and status indicators. The table is keyboard-navigable, and the data is presented in a logical order.

Compliance:

The table page complies with Janet regulations by ensuring proper data structuring, clear labeling, and responsive design. It also adheres to WCAG guidelines for accessibility.

Content Pages

Introduction:

The Content Pages for "Coral Reef Protection" and "Climate Impact" were designed to provide detailed information on these critical topics. Each page follows a structured layout with sections for introduction, importance, threats, conservation/mitigation strategies, and ways to support the cause. The pages are visually engaging, with images, horizontal navigation bars, and interactive elements to enhance user experience.

Technical Description

HTML Techniques:

Both content pages use semantic HTML tags like <header>, <main>, and <section> to structure the content. The pages are divided into sections using <section> tags, each with a unique ID for navigation. The horizontal navigation bar is created using an unordered list (<ul>) with anchor tags (<a>) for smooth scrolling between sections. Images are displayed using the <img> tag, and captions are added using <p> tags.

CSS Techniques:

The `content.css` file styles the pages with a consistent design. Key CSS features include:

  • Full-Width Rectangle – A container for images at the top of the page, styled with `display: flex;` and `gap: 2rem;` to create a responsive layout.
  • Horizontal Navigation Bar – Styled with `display: flex;` and `justify-content: space-around;` to align links horizontally.
  • Content Sections – Each section uses a semi-transparent background (`background-color: rgba(5, 25, 35, 0.8);`) and rounded corners (`border-radius: 10px;`) for a modern look.
  • Image Gallery – Images in the "Threats" and "Conservation/Mitigation" sections are displayed in a grid layout using `display: flex;` and `gap: 1rem;`.
  • Go to Top Button – A fixed-position button at the bottom-right corner allows users to quickly return to the top of the page.

Purpose and Logic:

The content pages are designed to educate users about coral reef protection and climate impact in an engaging and accessible way. The structured layout ensures that information is easy to navigate, while the use of images and interactive elements enhances user engagement. The responsive design ensures the pages work well on all devices.

Key HTML Tags and CSS Properties:

  • <section>, <nav>, <img>: Semantic structure and content.
  • display: flex;: Used for the image rectangle and horizontal navigation bar.
  • background-color: rgba();: Semi-transparent background for content sections.
  • position: fixed;: Used for the "Go to Top" button.

Challenges and Lessons Learned:

One challenge was ensuring the images in the full-width rectangle remained responsive without losing their aspect ratio. This was resolved using the `object-fit: cover;` property. Another challenge was implementing the horizontal navigation bar with smooth scrolling, which was achieved using anchor links and CSS styling. These experiences reinforced the importance of responsive design and user-friendly navigation.

Accessibility:

The content pages follow accessibility best practices by using semantic HTML tags and providing alt text for images. The navigation bar is keyboard-friendly, and the "Go to Top" button is clearly labeled for screen readers. The pages also use sufficient color contrast for readability.

Compliance:

The content pages comply with Janet regulations by ensuring proper content structuring, clear labeling, and responsive design. They also adhere to WCAG guidelines for accessibility.

Sitemap Page

Introduction:

The Sitemap Page provides an overview of the website's structure, including links to all major pages and sections. It uses a hierarchical layout to display the relationships between different pages, making it easy for users to navigate the site.

Technical Description

HTML Techniques:

The `sitemap.html` page uses a combination of <div> and <a> tags to create a hierarchical layout. Each node represents a page or section, and sub-pages are nested within their parent nodes. The page also includes links to editor and validation pages.

CSS Techniques:

The `sitemap.css` file styles the sitemap with a responsive grid layout, hover effects for nodes, and a clear visual hierarchy. The nodes are styled with background colors, border-radius, and box-shadow to create a visually appealing design.

Purpose and Logic:

The sitemap page is designed to provide a clear and concise overview of the website's structure. The hierarchical layout helps users understand the relationships between different pages, while the responsive design ensures accessibility on all devices.

Key HTML Tags and CSS Properties:

  • <div>, <a>: Hierarchical layout.
  • display: grid;: Responsive grid layout.
  • background-color: rgba();: Node styling.
  • box-shadow: 0 4px 8px rgba();: Visual hierarchy.

Challenges and Lessons Learned:

One challenge was ensuring the sitemap remained readable on smaller screens. This was resolved by using a responsive grid layout and adjusting font sizes. Another challenge was creating a clear visual hierarchy, which was achieved using background colors and box-shadow.

Accessibility:

The sitemap page follows accessibility best practices by using semantic HTML tags and providing clear labels for nodes. The page is keyboard-navigable, and the hierarchical layout is clearly labeled for screen readers.

Compliance:

The sitemap page complies with Janet regulations by ensuring proper content structuring, clear labeling, and responsive design. It also adheres to WCAG guidelines for accessibility.
